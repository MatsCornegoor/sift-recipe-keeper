# Contributing to Sift

Thanks for your interest in contributing! This document covers how to get involved.


## Ways to Contribute

- **Bug reports** — Found something broken? Open an issue with steps to reproduce.
- **Feature suggestions** — New ideas are always welcome, but Sift is intentionally minimalist. Features that add complexity without clear benefit to the core experience are unlikely to be accepted.
- **Model suggestions** — Found a model that works well with Sift? Open a PR to add it to the README.



## Pull Request Guidelines

> **Before opening a pull request, always open an issue first.** This ensures we can discuss the change and align on the approach before you invest time building it.

- Keep PRs focused — one change per PR makes review easier.
- Make sure the app builds and runs without errors before submitting.
